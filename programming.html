Analytics Dashboard

  • Python and html: I created this dashboard using Python with the Dash library. I also used embedded HTML to structure the graphs/tables which were created using Plotly. Pandas was used for reading data from multiple excel files and creating a dataframe.
  • Usability: The information at the top of the dashboard (seen in the first image) provides context for the user, such as the date range calculated from the excel files. I have included summary data as well. The graphs are also interactive - you can isolate one car to look at the price changes over time. You can also hover the mouse over any of the graphs and pie chart to get specific info.
  • Cleaning data from multiple excel files: Using Pandas I ensured the data was cleaned so that it could be manipulated and so the analysis made sense. Any overlapping data from the multiple excel sheets is managed by only using data from a sheet specified by the user. It is also able to work with null fields and blank fields.
  • Customer criteria and feedback: This dashboard was created using requests from the users of the web scraping script I developed (see below). I developed the dashboard to be able to answer questions they had about the data and insights they wanted to know. I also ensured to make all of these easy to understand in the dashboard. I iteratively released versions of the dashboard and collected feedback to make improvements.

Web Scraping Script

  • Python: I used Selenium for web scraping and Pandas for manipulating the data, and reading/writing excel sheets.
  • Usability: Based on feedback provided from pre-release versions of the script I ensured that the script is easy to use and the actions it takes are clearly presented to the user in the terminal, and any errors are presented in a user-friendly way. Details of these errors are stored in a file for troubleshooting. The aim of this script was to work in the background and not interrupt the work of the user, which was managed successfully by having the interface be a simple terminal which requires no interaction from the user. Any useful information is then presented through notifications or in an excel file. The user can also customise what data is scraped by entering details in a .txt file.

  • Writing data to an excel file: Once the data is collected into a dataframe, this can be used to write 4 sheets in a spreadsheet. The first is a table containing the prices for each car on each date that the script is ran. This is updated with every run of the script. This is then used to plot a graph of the prices for the cars over time. The lead times are also scraped and this is presented on a separate page. Finally, an overview page is created explaining the data, presenting the version of the script being used and shows the cheapest car, shortest lead time and which cars have newly been added or removed since the last scrape.

2D Platformer Game

  • Godot Game Engine: The Godot game engine and Godot's own programming language were used to create this game. I followed a YouTube tutorial to learn the basics and used their artwork to create the game. I could then move on to create my own features for the game.
  • Game Features: The character is able to move left and right and jump. If the character dies, the game automatically restarts. The player can collect coins, potions and collide with enemies, causing the player to die. The player wins by collecting all of the coins, which generates a message on screen.
  • Animations: The animations were created in the Godot game engine using multiple sprites to create an animation. For the character, there is a stationary animation, moving animation, jump animation and dying animation. For the coins, they have a rotating animation. Enemies have a walking animation.
